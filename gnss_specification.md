# Spresense GNSS サンプルプログラム 仕様書

本書は、Sony Spresense SDK に同梱されている GNSS (GPS) サンプルプログラム（`examples/gnss/gnss_poll_main.c`）の動作仕様についてまとめたドキュメントです。

## 1. プログラム概要
本プログラムは、Spresense に内蔵されている GNSS（全地球測位システム）モジュールを制御し、現在の緯度・経度を取得してシリアルコンソールに出力するサンプルアプリケーションです。
デバイスファイルから非同期にデータを受信するため、`poll()` システムコールを利用して効率的に待機・読み取りを行います。

## 2. 動作シーケンス
本プログラムを実行すると、以下の順序で処理が行われます。

1. **デバイスのオープン**: 
   - GNSS デバイスファイル（通常 `/dev/gps`）を読み取り専用 (`O_RDONLY`) でオープンします。
2. **GNSS パラメータ設定**:
   - `ioctl` コマンドを用いて、以下の動作パラメータを設定します。
     - 測位サイクル: **1000 msec** （1秒間隔）
     - 使用する衛星システム: **GPS** および **GLONASS**
3. **測位開始**:
   - **Hot Start モード** (`CXD56_GNSS_STMOD_HOT`) で測位処理を開始します。
4. **測位ループ処理**:
   - `poll()` 関数で GNSS からのデータ通知を待機します。
   - 通知を受信すると `read()` を実行し、取得したデータ（時刻や緯度・経度）をコンソールに出力します。
   - **終了条件**: 測位が確立（Fix）してから、**200回（約200秒間）** のデータ出力を完了するまでループを継続します。この長めの時間は、次回以降の Hot Start を速くするために衛星の軌道情報（エフェメリス）を十分に蓄積するための仕様です。
5. **GNSS の停止と終了**:
   - 規定回数に達すると測位処理をストップし、デバイスファイルを閉じてプログラムを終了します。

## 3. 出力フォーマット
シリアルコンソールへの出力は、以下のフォーマットで行われます。

### 3.1 初期化時の出力
```text
Hello, GNSS <<USE POLL>> SAMPLE!!
start GNSS OK
```

### 3.2 測位データ出力（毎秒）
GNSS デバイスからデータを受け取るたびに、まず共通して時刻が出力されます。
```text
>Hour:[時], minute:[分], sec:[秒], usec:[マイクロ秒]
```

#### 測位未完了（衛星探索中）の場合
位置情報が確定していない場合は、時刻の後に以下のメッセージが出力されます。
```text
>No Positioning Data
```

#### 測位完了（Fix）の場合
位置情報が確定（2D Fix または 3D Fix）した場合は、緯度と経度が **度・分・秒（の小数）** の DMF 形式に変換されて出力されます。
```text
>LAT [度].[分].[分の小数部分]
>LNG [度].[分].[分の小数部分]
```
*(例: `>LAT 35.40.8741`)*

## 4. プログラムの中断と制限事項
- 室内などの空が見えない場所では衛星の電波を受信できないため、ループ処理が延々と `No Positioning Data` を出力し続けます。
- 初回の測位（実質的なコールドスタート）には、通常屋外で1〜3分程度かかる場合があります。

## 5. 対象ソースコード
- **ソースファイル**: `spresense/examples/gnss/gnss_poll_main.c`
- **主要なシステムコール**: `open()`, `ioctl()`, `poll()`, `read()`, `close()`
